# PRODUCT_CONTEXT.md

本文件记录 Awakening 觉醒计划及相关 B 端 / C 端实验的产品上下文。

## 产品群关系

已知事实：

- 本项目包含至少三类目标：B 端建筑行业实验、C 端 Awakening 觉醒计划、个人 IP 叙事。
- 这些目标共同服务长期命题：AI 时代，个体和组织如何完成重构。

判断：

- 当前最需要避免的是把“宏大叙事”误当作“产品需求”。
- 产品工作应先围绕真实用户、真实场景、真实转化行为展开。
- B 端和 C 端应该分别验证，避免混成一个无法判断成败的大项目。

## C 端：Awakening 觉醒计划

已知事实：

- 目标方向：验证年轻人 AI 职业重构需求、内容传播和用户转化。

待验证假设：

- 年轻人不只是想了解 AI，而是需要一套可执行的职业重构路径。
- 用户愿意为“AI 能力提升 + 职业方向重构 + 现实任务陪跑”付费。
- 内容传播可以带来足够高质量的用户线索。

待确认：

- 核心用户是谁：大学生、应届生、转行者、职场新人或其他。
- 用户最强痛点是什么：迷茫、就业压力、技能焦虑、信息过载、执行力不足或其他。
- 当前最小产品形态是什么：文章、短视频、社群、训练营、咨询、工具或课程。
- 当前转化路径是什么。
- 当前关键指标是什么。

## B 端：建筑行业 AI 工作流改造

已知事实：

- B 端实验从建筑行业切入。
- 目标方向：验证传统行业真实 AI 工作流改造机会。

待验证假设：

- 建筑行业存在大量重复性文档、沟通、方案、报价、资料整理等 AI 可改造环节。
- 传统行业客户更愿意为“节省人力 / 缩短周期 / 降低错误率”的具体结果付费，而不是为“AI 概念”付费。

待确认：

- 具体细分场景：设计院、施工单位、造价、招投标、项目管理、材料商或其他。
- 当前客户触达渠道。
- 是否有真实访谈记录。
- 是否有试点工作流。
- 是否有报价、交付边界和验收标准。

## 个人 IP 与内容

已知事实：

- 个人 IP 叙事是记录软件工程学生探索 AI 时代个体与组织重构的过程。

判断：

- 内容应优先记录真实实验，而不是只输出观点。
- 每条内容最好连接一个具体动作：访谈、测试、复盘、产品迭代、学习进度或用户反馈。

待确认：

- 首选内容平台。
- 内容节奏。
- 内容栏目。
- 是否有统一的转化入口。
